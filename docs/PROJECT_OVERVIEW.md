# 프로젝트 개요 (`PROJECT_OVERVIEW.md`)

이 프로젝트는 React + TypeScript + Vite 환경에서 간단한 Notion 형태의 문서 관리 도구를 구축하면서, 프론트엔드의 기초 아키텍처와 상태 변화 흐름을 깊이 있게 이해하기 위해 기획된 학습용 프로젝트입니다. 

---

## 1. 프로젝트 목적 및 학습 목표

### 프로젝트의 주요 지향점
* Notion의 복잡한 서식이나 디자인(드래그 앤 드롭, 블록 기반 에디터)을 똑같이 모사하는 것이 목적이 아닙니다.
* **데이터가 화면에 그려지고, 사용자의 동작에 따라 흐르는 핵심 구조**를 학습하는 것이 최우선 목표입니다.

### 핵심 학습 목표
1. **컴포넌트 설계 기준**: React 프로젝트에서 화면을 컴포넌트 단위로 분리하는 명확한 기준 확립.
2. **역할 분담 및 협업**: `Page`, `Layout`, `Header`, `Sidebar`, `Component`, `Hook`, `API`, `Type`이 어떻게 서로 맞물려 돌아가는지 이해.
3. **단방향 데이터 흐름**: 사용자의 이벤트 발생 → 백엔드 API 요청 → 컴포넌트 상태 갱신 → 화면 재렌더링의 생명 주기를 직접 눈으로 확인.
4. **React Core API 실무 적용**: `useState`, `useEffect`, `Props`, `Event Handler`의 명확한 차이와 비동기 처리에 대한 적재적소 적용.
5. **AI 주도 코딩 탈피**: AI가 작성한 코드를 통째로 복사-붙여넣기 하는 대신, 제공된 단계별 유도 코드에 맞춰 개발자가 직접 타이핑하며 코드 흐름을 완전히 체득.
6. **기능 구현 로드맵 체화**: 기능 하나를 완성하기 위해 어떤 파일들을 만들고, 어떤 순서로 구현해 나가야 하는지 학습.

## 2. 구현 범위 및 기술 사양

### 2.1. 구현 범위 (핵심 기능)
* **사이드바 문서 목록**: 데이터베이스(Mock 또는 실 서버)에 등록된 문서의 제목 목록을 왼쪽에 렌더링.
* **문서 선택 및 상세 조회**: 특정 문서를 클릭하면 우측 메인 영역에 제목과 본문이 상세 출력됨.
* **문서 생성**: '새 페이지' 버튼을 누르면 즉시 새로운 문서가 목록에 추가되고 작성 화면으로 전환됨.
* **제목 및 본문 수정**: 제목 `<input>`과 본문 `<textarea>`를 통해 입력 내용을 실시간 반영.
* **문서 수동/자동 저장**: 저장 버튼을 통한 수동 API 전송 및 디바운싱(Debouncing) 기술을 이용한 자동 저장 기능.
* **문서 삭제**: 삭제 버튼을 눌러 목록과 상세 화면에서 데이터를 즉시 제거.
* **상태 피드백**: 데이터를 불러오는 로딩 상태(`loading`) 및 네트워크/서버 에러 상태(`error`) 화면 표시.

### 2.2. 제외 범위 (학습 효율을 위한 축소)
* 블록 기반 에디터 (예: Slash command `/`, Rich text editor) -> 일반 Input과 Textarea로 대체.
* 드래그 앤 드롭을 통한 문서 계층 구조 이동.
* 다중 협업 기능 (실시간 동시 편집 및 소켓 통신).
* 복잡한 전환 애니메이션 및 스타일 모션.

### 2.3. 기술 스택
* **프론트엔드 라이브러리**: React 18+ (TypeScript 적용)
* **빌드 도구**: Vite
* **스타일링**: Vanilla CSS (CSS Variables를 활용한 프리미엄 테마)
* **HTTP 클라이언트**: Axios (Spring Boot API 연동 대비)
* **라우터**: React Router DOM (Workspace 라우팅)
* **상태 관리**: 컴포넌트 로컬 상태(`useState`) 및 상태 끌어올리기(State Lifting) 적용 (초기에는 전역 상태 라이브러리 도입을 지양하여 원리를 먼저 파악)

## 4. 최종 완성의 기준
1. **기능적 기준**: 사이드바 문서 목록 클릭 시 상세 내용이 로드되며, 수정 후 저장/자동저장 및 삭제가 정상 작동함. API 처리 도중 로딩 스피너와 네트워크 에러 상황에 대한 안내가 화면에 표현되어야 함.
2. **학습적 기준**: 학습자가 [LEARNING_CHECKLIST.md](file:///c:/Users/Dalyume/Documents/DAL/Coding/notion-clone/docs/LEARNING_CHECKLIST.md)의 10가지 이상의 핵심 개념 질문에 대하여 소스 코드 라인 번호를 짚어가며 직접 설명할 수 있어야 함.
3. **코드적 기준**: AI가 생성한 완성본이 아니라, 주석과 분리된 컴포넌트 구조를 설계 규약에 맞게 작성하고 정상적인 TypeScript 컴파일 테스트를 통과해야 함.
